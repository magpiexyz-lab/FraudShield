# UI: shadcn/ui
> Used when experiment.yaml has `stack.ui: shadcn`

## Packages (devDependencies)
```bash
npm install -D tailwindcss @tailwindcss/postcss
```

## Setup Commands
Run in order:
1. Create prerequisites that `shadcn init` expects to exist (Tailwind CSS v4):
   - `postcss.config.mjs`:
     ```js
     const config = { plugins: { "@tailwindcss/postcss": {} } };
     export default config;
     ```
   - `src/app/globals.css`:
     ```css
     @import "tailwindcss";
     ```
2. `npx shadcn@latest init --yes --defaults` — configures component paths and theme
3. `npx shadcn@latest add -y button input card label separator badge tabs accordion avatar tooltip dialog alert progress` — install base components (add more later as needed)

## Post-Setup Verification
After running setup commands, verify:
- `postcss.config.mjs` uses `@tailwindcss/postcss` — fix if shadcn generated a different configuration
- `src/app/globals.css` contains `@import "tailwindcss"` — fix if missing
- `package.json` still has `dev`, `build`, `start`, and `lint` scripts — re-add if shadcn overwrote them
- `eslint.config.mjs` contains the `no-restricted-syntax` rule banning `asChild` — re-add if shadcn init overwrote it

## Theme Setup

After `shadcn init` completes and post-setup checks pass, apply these customizations:

### 1. Font (in layout.tsx — created in Step 3)
Import the chosen font(s) from `next/font/google` (ships with Next.js — no extra package). Choose a font that matches the product domain — distinctive and intentional, not generic.
- Example: `const displayFont = YourChosenFont({ subsets: ["latin"], variable: "--font-sans" })`
- Apply `className={displayFont.variable}` to the `<html>` tag
- If using a display + body font pair, apply both variables: `className={\`${displayFont.variable} ${bodyFont.variable}\`}`

### 2. CSS overrides (in globals.css — exists after shadcn init)
Override CSS custom properties to match the product domain. There are no hardcoded default values — choose a color palette, border radius, and other theme tokens that feel distinctive and intentional.

### 3. Post-theme verification
- globals.css contains `--primary` override
- layout.tsx imports the display font and applies variable to `<html>`

## Patterns
- Always use library components (`<Button>`, `<Input>`, `<Card>`, etc.) — don't mix with raw HTML elements (`<button>`, `<input>`)
- Follow `.claude/patterns/design.md` for quality invariants.
- Import components from `@/components/ui/<component>` — these are auto-generated by shadcn
- When a new component is needed, add it with `npx shadcn@latest add -y <component>` rather than building from scratch

## v4 Breaking Changes

**`asChild` removed**: The Radix `asChild` composition pattern is not supported in shadcn v4. For link-styled buttons, use `buttonVariants()` directly on the element:

```tsx
import Link from "next/link";
import { Button } from "@/components/ui/button";
import { buttonVariants } from "@/components/ui/button";

// WRONG (v4 does not support asChild):
<Button asChild><Link href="/foo">Go</Link></Button>

// CORRECT:
<Link href="/foo" className={buttonVariants({ variant: "outline" })}>Go</Link>
```

This applies to all Radix-based components: `DialogTrigger`, `TooltipTrigger`, `DropdownMenuTrigger`, etc. Wrap children directly instead of using `asChild`.

**Trigger + interactive element**: When a trigger component (`TooltipTrigger`, `DialogTrigger`, etc.) wraps an interactive element like `<Button>`, use the `render` prop to avoid nested `<button>` elements (HTML validity violation):

```tsx
import { Button } from "@/components/ui/button";
import { TooltipTrigger } from "@/components/ui/tooltip";

// WRONG (nested <button> — invalid HTML):
<TooltipTrigger><Button>Click</Button></TooltipTrigger>

// CORRECT (render prop composes without nesting):
<TooltipTrigger render={<Button variant="ghost" />}>
  Click
</TooltipTrigger>
```

The `render` prop replaces the trigger's default `<button>` wrapper with the provided element, passing through all trigger behavior (event handlers, aria attributes). Children of the trigger become children of the rendered element.
